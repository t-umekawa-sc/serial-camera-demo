# 製造番号カメラデモ プロジェクト現況

最終更新: 2026-06-30

## 概要

スマートフォンのブラウザでカメラを起動し、画面上の赤いガイド枠に合わせた製造番号部分だけを切り出して撮影・蓄積し、最終的にサーバへ一括アップロードすることを想定したWebデモアプリケーション。

現段階は **通常Web版（フロントエンドのみ）** で、アップロード処理はダミー実装。サーバ側のOCR連携は未着手。

## 技術構成

- 純粋な静的Webアプリ（HTML / CSS / 素のJavaScript）。サーバ処理は不要
- ビルドツール・フレームワークは不使用
- バーコード読み取りに **ZXing（`@zxing/library`）** をCDNから読み込み（唯一の外部ライブラリ）
- カメラ操作は自前の `getUserMedia`（背面カメラ優先、1920×1080を要求）。バーコードは「バーコード読取（長押し）」ボタンを押している間、ガイド枠内を約200ms間隔で連続解析（ZXing `MultiFormatReader`）。読めた時点で成功し停止、指を離すまでに読めなければエラー
- **読取種別の切り替え**: 「読取種別：バーコード／QR」トグルで、解析対象フォーマットを1次元バーコード（CODE_128/39/93・ITF・EAN・UPC・CODABAR）と2次元コード（QR・DataMatrix・Aztec・PDF417）に切り替える。ZXingはモードに含まれないフォーマットを解析しないため、バーコードとQRが近接印字されていても狙った方だけを読める（既定はバーコード）
- 各フレームは「素の画像 → 失敗時のみ影補正 → さらに失敗時は反転（ネガ）」の三段で解析。影補正は背景の明るさ分布（縮小描画でぼかして推定）で割って照明ムラを平坦化し、軽くコントラスト強調する処理（固定影に有効）。反転は黒地に白バー等の反転印字に対応。いずれも素画像を先に試すため、きれいなラベルでは速度・精度を落とさない
- 撮影とバーコード読み取りは同じガイド枠の切り出し座標（`getGuideSourceRect`）を共有
- 画像の切り出しは Canvas（`drawImage` + `toBlob`、JPEG品質0.9）
- `index.html` の `style.css` / `app.js` 参照にバージョン文字列（`?v=2`）を付与し、更新時のキャッシュ不一致を防止

## 実装済み機能

- **カメラ起動**: 起動時に自動でカメラを開始（ZXing経由）。失敗時はステータス表示とアラートで通知
- **ガイド枠内のバーコード読み取り（長押し連続試行）**: 「バーコード読取（長押し）」ボタンを押している間、赤いガイド枠内を連続で読み取り試行。影で一部フレームが失敗しても、押している間に読めたフレームで成功にできる。成功時は検出値と種別をバナー表示（対応端末ではバイブ通知）
- **読取種別の切り替え（バーコード／QR）**: トグルボタンで解析対象を1次元バーコードと2次元コード（QR等）に切り替え。近接印字されたQRに引っ張られず狙ったコードだけを読む狙い（既定はバーコード）。読み取り中は切り替え不可
- **読取対象バーコードの複数選択**: 「読取対象」パネルで1次元バーコード9種（CODE_128/39/93・CODABAR・ITF・EAN-13/8・UPC-A/E）をチェックボックスで個別ON/OFF。誤読しやすいCODABAR・ITFを外すなど用途に合わせて絞れる。選択は `localStorage` に保持し次回起動時も維持。QRモード時は無効（QR側は固定フォーマット）
- **影補正の前処理**: 連続試行の各フレームで、素の画像で読めない場合に照明ムラ補正＋コントラスト強調をかけた画像で再解析。倉庫などの固定影に対する読み取り率を底上げ（テカリ・白飛び・ブレには効果なし）
- **反転（ネガ）画像の追試**: 影補正でも読めない場合、白黒を反転した画像でも解析。黒地に白バー等の反転印字に対応
- **読み取り失敗時のエラー表示**: 指を離すまでに枠内のバーコードを検出できなかった場合、同じバナーを赤色にして「読み取り失敗」と調整を促すメッセージを表示
- **ガイド枠撮影**: 画面の赤枠と同じ範囲を映像解像度に合わせて切り出し、メモリ上に保持
- **撮影ガイドの補助表示**: S/Nの先頭を合わせる開始マーカー（左端から内側にインセット）と、傾き防止の水平基準線を枠内に表示
- **撮り方のコツ表示**: 精度を上げる撮影のポイント（先頭/水平合わせ、近づく、ピント、明るさ・反射回避、正面撮影など）をパネルで表示。初回起動時に自動表示し、「コツ」ボタンでいつでも再表示可能
- **連続撮影**: 撮影枚数をカウント表示し、複数枚を蓄積
- **画像ID付与**: `YYYYMMDD_HHMMSS_連番` 形式のIDを自動生成
- **一覧確認パネル**: 撮影済み画像をサムネイル・撮影日時・IDで一覧表示
- **個別削除**: 一覧から画像を削除（`URL.revokeObjectURL` でメモリ解放）
- **一括アップロード（ダミー）**: `FormData` を組み立てるがアラート表示のみ。実送信は未実装
- **画面向き対応**: 縦・横（landscape）でレイアウトを切り替え

## 未実装 / 今後の予定

- **サーバへの実アップロード**: 現状 `app.js` 内にコメントとしてFastAPIへの送信例を記載のみ
- **サーバ側OCR処理**: Python + OpenCV + PaddleOCR で製造番号を読み取る想定（README記載の次ステップ）
- **永続化**: 撮影画像はメモリ保持のみで、リロードすると消える
- **バーコード値と撮影画像の紐づけ**: 現状は読み取り値を表示するのみ。撮影画像やアップロードのメタデータに付与する連携は未実装
- **読み取り範囲**: バーコードは赤いガイド枠内のみを対象に読み取る（映像全体ではない）。枠外のバーコードは読まれない

### スマートフォン

## 留意点

- アップロードはダミーのため、撮影データは外部に送信されない
- 画像はすべてブラウザのメモリ上にのみ存在し、タブを閉じる／リロードで失われる
- カメラ権限・HTTPS要件があるため、`file://` 直開きやHTTP接続では動作しないことがある
- ガイド枠はS/Nの長さが不明な前提で広め（幅80%）に取り、クロップは枠全体を切り出す（右端の欠けを防ぐ）。開始マーカーは左端に密着させず、OCR用の余白を確保するためインセットしている。実機で過不足が分かり次第、枠幅の調整が望ましい
